Attribute und Daten

Wichtig

Die in der XML-Datei angelegten Attribute und Elemente sind nicht frei wählbar, sondern orientieren sich an einer Schema-Definition. Diese gibt vor, welche Elemente und Attribute verwendet werden dürfen. Die Schema Definition für die Testtakers.xml ist hier ersichtlich.

Die Schema-Definition wird im Kopf der XML eingebunden:

<Testtakers xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="https://raw.githubusercontent.com/iqb-berlin/testcenter/15.0.1/definitions/vo_Testtakers.xsd">

</Testtakers>

Nachfolgend ist eine beispielhafte Testtaker-XML zu sehen:

<?xml version="1.0" encoding="utf-8"?>
<Testtakers xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="https://raw.githubusercontent.com/iqb-berlin/testcenter/15.0.1/definitions/vo_Testtakers.xsd">

<Metadata>
    <Description>
    This file contains some logins for testing and works a a sample for developers.
    </Description>
</Metadata>

<CustomTexts>
    <CustomText key="app_title">Hier steht ein Custom Text</CustomText>
</CustomTexts>

<Group id="sample_group" validFrom="1/3/2020 10:00" label="Primary Sample Group">
    <Login mode="run-hot-return" name="test" pw="user123">
        <Booklet codes="xxx yyy">BOOKLET.SAMPLE-1</Booklet>
        <Booklet codes="xxx yyy">BOOKLET.SAMPLE-3</Booklet>
        <Booklet>BOOKLET.SAMPLE-2</Booklet>
    </Login>
    <Login mode="monitor-group" name="test-group-monitor" pw="user123"/>
</Group>

<Group id="study_group" label="A group for the study monitor">
    <!-- the study monitor has access to all other groups on the workspace -->
    <Login mode="monitor-study" name="test-study-monitor" pw="user123"/>
</Group>

<Group id="trial_group" label="A Group for Trials and Demos" validFor="45">
    <Login mode="run-trial" name="test-trial" pw="user123">
        <Booklet>BOOKLET.SAMPLE-1</Booklet>
    </Login>
    <Login mode="run-demo" name="test-demo" pw="user123">
        <Booklet>BOOKLET.SAMPLE-1</Booklet>
    </Login>
    <Login mode="run-simulation" name="test-simulation" pw="user123">
        <Booklet>BOOKLET.SAMPLE-1</Booklet>
    </Login>
</Group>

<Group id="passwordless_group" label="A group of persons without password">
    <Login mode="run-hot-restart" name="test-no-pw">
        <Booklet>BOOKLET.SAMPLE-1</Booklet>
    </Login>
    <Login mode="run-trial" name="test-no-pw-trial">
        <Booklet>BOOKLET.SAMPLE-1</Booklet>
    </Login>
</Group>

<Group id="expired_group" validTo="1/3/2020 19:30" label="An already expired group">
    <Login mode="run-hot-restart" name="test-expired">
        <Booklet>BOOKLET.SAMPLE-1</Booklet>
    </Login>
    <Login mode="monitor-group" name="expired-group-monitor" pw="user123"/>
    <Login mode="monitor-study" name="expired-study-monitor" pw="user123"/>
</Group>

<Group id="future_group" validFrom="26/3/2030 9:00" label="An not yet active group">
    <Login mode="run-hot-restart" name="test-future">
        <Booklet>BOOKLET.SAMPLE-1</Booklet>
    </Login>
</Group>

</Testtakers>

Die verfügbaren Attribute und die zugehörigen möglichen Werte und deren Bedeutung werden nachfolgend aufgelistet. Die Attribute und Werte werden dabei dem jeweiligen Xml-Hauptelement zugeordnet.

Generelle Test-Konfiguartion

Die nachfolgenden Elemente und deren verfügbare Attribute realisieren die folgende Funktionen:

  • Anlegen der Zugriffsdaten (Logins)
  • Modi der Testdurchführung
  • Gruppenzuweisungen
  • Testheftzuweisungen
  • Gruppenbeschränkungen (Bsp.: Logins einer Gruppe sind erst ab einer bestimmten Zeit gültig)

<Metadata>

Description >xxx< Datum frei wählbarer Text für die Metadaten Beschreibung

<Group>

Group id Attribut frei wählbarer Wert
Group label Attribut frei wählbarer Wert
Group validTo Attribut Bestimmt wie lange ein Zugang gültig ist. Frei wählbarer Wert im Format:
dd/mm/yyyy hh:mm
Group validFrom Attribut Bestimmt ab wann ein Zugang gültig ist. Frei wählbarer Wert im Format:
dd/mm/yyyy hh:mm
Group validFor Attribut Bestimmt einen Zeitraum für einen Zugang. Frei wählbarer Minutenwert im Format: Integer
Login name Attribut Frei wählbarer Wert. Wird zur Anmeldung am Testcenter benötigt, um entsprechend des gewählten Testmodi einen Test durchzuführen.
Login mode Attribut Vordefinierter Wert. Bestimmt wie ein Test ablaufen soll (finaler Test oder Probetest etc.). Weiteres ist den Referenzen: Modi der Testdurchführung zu entnehmen
Login pw Attribut Frei wählbares Passwort. Wird zur Anmeldung am Testcenter benötigt, um entsprechend des gewählten Testmodi einen Test durchzuführen.
Booklet >xxx< Datum Frei wählbarer Wert. Die ID in der Booklet.xml muss den selben Namen tragen (Groß-/Kleinschreibung beachten!)

Textersetzungen

Diese Anwendung ermöglicht es zur Laufzeit Änderungen an Texten durchzuführen und somit individuelle Texte zu schaffen. Welche Textteile innerhalb der Anwendung ersetzt werden sollen, wird mit den Attributen des nachfolgenden Elements festgelegt.

<CustomText>

CustomText key Attribut Vordefinierter Wert. Erzeugt einen Custom Text. Siehe Custom Text Configuration.
CustomText >xxx< Datum frei wählbar Text zur Anzeige eines Custom Text
Hinweis

Der Attributwert für das Attribut key ist nicht frei wählbar. Es können nur Werte verwendet werden, die hier gelistet sind! Alle Texte, die nicht geändert werden, erhalten den Standardtext.

Wichtig

Achten Sie bei der Verwendung auf die richtige Schreibweise und beachten Sie die Groß-/Kleinschreibung.

Einrichten der Testleitungskonsole

Die Testleitung kann in der Testtaker.xml über den Login: Monitor-Group (mehr Informationen zum Thema Modi der Testdurchführung finden Sie hier festlegen welche Gruppen der Testleitkonsole hinzugefügt werden sollen. Dazu wird in der entsprechenden Gruppe ein Login: Monitor-Group angelegt. Nachfolgend finden Sie zwei Beispiele wie Gruppen angelegt werden können:

Ohne Passwort. Sinnvoll wenn nur Links an die Testpersonen versendet werden.

Jede Testperson erhält ein eigenes Passwort. Dies stellt die gängigste Methode dar.
